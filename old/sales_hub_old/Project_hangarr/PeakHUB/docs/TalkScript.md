# TalkScript（PeakHub）— 0) 最初の15秒

## 目的

- キーマンに繋がった直後に「何の電話か」を最短で伝え、質問1つに入る
- 抽象語を避け、資料のルール（誇張NG）に準拠する

## 0-1. キーマン接続直後（15秒スクリプト）

### パターンA（基本）

1. 謝罪・名乗り
   - 「突然すみません、PeakHubの◯◯です」
2. 要件（採用の構造）
   - 「採用の件で、**人材紹介中心になりがちな状況から、媒体やスカウトも含めて配分を整えるRPO**をご支援してまして」
3. 時間の許可取り
   - 「**15秒だけ**確認させてください」
4. 次の一問（そのまま続ける）
   - 「今の採用手法って、紹介・媒体・スカウトだと、どれが中心ですか」

### パターンB（“工数”から入る）

- 「突然すみません、PeakHubの◯◯です。採用の件で、**人事の採用実務を巻き取って工数を減らす**支援をしてまして、**15秒だけ**確認させてください。今、採用業務って兼務だったり少人数で回されてますか」

### パターンC（既存RPO/代行の不満に刺す）

- 「突然すみません、PeakHubの◯◯です。採用の件で、**社内にノウハウやマニュアルを残して“卒業できるRPO”**のご支援をしてまして、**15秒だけ**確認させてください。今、RPOや代行って既にご利用されてますか」

## 0-2. 受付突破→キーマン接続まで（前段の一言）

- 「お忙しいところすみません。弊社、電子機器や製造・鋳造などの専門性の高い皆様の、現場～営業様の採用を専門にしております。内部「」」」


## NG（口に出さない）

- 「採用コストを◯%削減できます」（NG）→「最適化」を使う
- 「成功率◯%」の保証（NG）→事例ベースのみ
- 「独自AI/独自DB」（NG）→「Claude等のAIを活用したワークフロー」の範囲に留める
- 「初回2週間無料」（アポ獲得時に出さない）
