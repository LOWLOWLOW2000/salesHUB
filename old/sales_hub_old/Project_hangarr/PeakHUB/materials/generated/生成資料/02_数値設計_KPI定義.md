# PeakHUB コールドコール戦略・戦術 — 数値設計（KPI定義）

## 目的
- 日報・週報・社内報告でKPIをブレなく集計する
- 施策（リスト/トーク/分岐）と成果指標を紐づけて改善を回す

## コアKPI（PJシートの運用に合わせる）
### 架電件数
- 定義: 発信総数
- 注意: 不通/留守電/折り返し依頼を含めるかは運用で固定する（混ぜない）

### 受付NG数
- 定義: 受付段階で終了した件数（キーマン接続に至らず）
- 推奨の理由分類: 担当不在 / 取り次ぎ不可 / 迷惑扱い / 情報不足 / その他

### キーマン接続数
- 定義: 意思決定者または採用責任者（採用の意思決定に影響する役職）と会話できた件数
- 注意: “担当者（受付が指す担当）”と“キーマン”を混同しない

### キーマンNG数
- 定義: キーマン接続後に終了した件数（次アクションを獲得できず）
- 推奨の理由分類: 現状満足 / 予算なし / 時期未定 / 既存RPOあり / 紹介で十分 / その他

### 資料送付数
- 定義: 送付先（メール等）が確定し、送付が完了した件数
- 注意: 「送っていい」だけで送付未完了はカウントしない

### アポイント数
- 定義: 日時・形式（オンライン/訪問）が確定した件数
- 注意: “検討するので資料”はアポに含めない

## 派生率（改善の指標）
### 受付突破率
- 式: キーマン接続数 / 架電件数

### 次アクション率
- 式: (資料送付数 + アポイント数) / キーマン接続数

### アポ率（キーマン接続起点）
- 式: アポイント数 / キーマン接続数

## 推奨の週次レビュー順（迷わないための固定手順）
1. 受付突破率（リスト×受付ワードの問題切り分け）
2. 次アクション率（15秒スクリプト×分岐の問題切り分け）
3. アポ率（提案の刺さり×反論処理の問題切り分け）
4. NG理由Top3（翌週の実験テーマに直結）
