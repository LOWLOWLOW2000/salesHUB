# 投入用CSV仕様（PeakHUB PJシート準拠）

- **対象シート**: PJシート > 架電リスト（投入）
- **列数**: 95列（A〜CQ）
- **エンコード**: UTF-8 BOM付き（`utf-8-sig`）
- **区切り**: カンマ `,`
- **CQ以降（CR〜）**: 関数列のため **CSVに含めない・上書きしない**
- **貼り付け先セル**: A2（値のみ貼り付け）

## 列ブロック定義

| 列 | ブロック | 備考 |
|---|---|---|
| A〜D | 基本情報① | 郵便番号〜重複フラグ |
| E | `_ignore` | PJシート上はクライアントのNO欄。CSV投入時は**常に空** |
| F〜X | 基本情報② | リストソース〜検索URL |
| Y | 従業員数増減6ヶ月 | |
| Z | `lead_id` | 旧「予備①」を流用。`PEAK-000001` 形式 |
| AA〜AH | 予備②〜⑨ | 投入時は常に空 |
| AI〜AQ | 担当・ステータス | 役職〜最新架電日。投入時は常に空 |
| AR〜CQ | 架電結果①〜⑩ | 5列×10ブロック。投入時は常に空 |
| CR〜 | **CSVに含めない** | PJシート側の関数列 |

## 投入ルール

### 必須（空は禁止）
- `lead_id` / `リストソース` / `IS担当者` / `企業名` / `電話番号` / `都道府県` / `市区町村`

### 推奨（取得できていれば埋める）
- `郵便番号` / `被保険者数` / `住所` / `HP` / `事業内容` / `サービス内容`
- `代表者名` / `上場区分` / `資本金` / `売上高` / `事業タグ` / `部署情報` / `拠点情報`
- `メール` / `従業員数増減6ヶ月`

### デフォルト値
- `アプローチ不可` → `FALSE`
- `重複フラグ` → 空（PJシート側の関数で判定）

### 常に空
- `_ignore` / `検索URL`（PJシートのHYPERLINK式を保護）
- `予備②〜⑨` / `役職`〜`最新架電日` / `架電日①〜NG理由⑩`

---

## 正規化ルール

| 列 | ルール |
|---|---|
| `電話番号` | ハイフンなし・先頭`0`保持・ダブルクォート囲み（例: `"0744276787"`） |
| `郵便番号` | `123-4567` 形式（ハイフンあり） |
| `資本金` / `売上高` | カンマなし整数（例: `30000000`） |
| `事業内容` / `サービス内容` | 改行 → 半角スペースに置換 |
| `検索URL` | 常に空（HYPERLINK式保護） |
| `HP` | `https://` または `http://` で始まる完全URL |
| `_ignore` | 常に空 |

---

## lead_id 採番ルール

- 形式: `PEAK-` + 6桁ゼロ埋め連番（例: `PEAK-000001`）
- **一度発番したら変更しない**（架電結果・レポートとの突合キー）
- 重複検知キー（投入時チェック）:
  - 第1キー: `電話番号`（正規化後）
  - 第2キー: `企業名` + `都道府県`（正規化後）
